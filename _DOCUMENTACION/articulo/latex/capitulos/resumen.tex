\begin{abstract}
    
    Este artículo investiga sobre la viabilidad de utilizar modelos de razonamiento automático para la generación de código fuente mantenible. Para ello, sostiene dos tesis fundamentales: en primer lugar, es necesaria la aplicación de modelos estrictos y restringidos al ámbito específico del software a generar, estructurando tanto la fase de análisis como la de diseño para facilitar su interpretación por parte de la inteligencia artificial. En segundo lugar, debemos alejarnos de la estereotipación de lenguajes de modelado genéricos, como UML, priorizando el uso de Lenguajes de Dominio Específico (DSL) propios del ámbito de aplicación (entendiendo ámbito de aplicación ejemplos como la generación de software, gestión de pedidos, comercio electrónico, etc), independientes de la implementación técnica pero estrechamente ligados al dominio del problema. A partir de estos preceptos, el artículo toma como marco metodológico la Arquitectura Dirigida por Modelos (MDA), y como ámbito ejemplo el desarrollo de software para sintetizadores de audio. Como resultado, se ha diseñado un conjunto de DSL y un Entorno de Desarrollo Integrado (IDE) denominado MDAA (\textit{Model Driven Development Audio}), capaces de abstraer el conocimiento del dominio y transmitir información altamente estructurada a un LLM genérico, permitiendo así la generación automática del código fuente objetivo, manteniendo a su vez la trazabilidad entre los distintos niveles de abstracción.

    \vspace{0.5em}
    \noindent\textbf{Palabras clave---} MDA, LLM, Generación automática de código, DSL, Síntesis de audio, Ingeniería de Software.
\end{abstract}
